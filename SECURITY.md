# Security policy

DELETE-DENIED protects the supported Codex `PreToolUse` shell-hook path. It is not an
OS-wide filesystem sandbox.

## Report privately

Use GitHub Security Advisories: [Report a vulnerability](https://github.com/Seokwoooo/DELETE-DENIED/security/advisories/new).
Include the affected release, operating system, architecture, Codex version, observed
output, and the smallest useful description of the issue. Do not include private files,
credentials, or other user data.

If the advisory form is unavailable, open a non-sensitive issue asking for a private
reporting channel.

## Supported release

The supported release line is v0.2.x for 64-bit macOS and Windows.

Report SHA-256 mismatches, failure to preserve unrelated `hooks.json` entries, or removal
of files outside the current user's DELETE-DENIED directory.
